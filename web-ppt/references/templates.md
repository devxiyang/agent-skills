# Slide Templates by Scenario

## Project / Work Report

Structure: Context → Problem → Solution → Results → Next Steps

```
Slide 1  layout-cover    Project name, date, your name
Slide 2  layout-title    "Background"
Slide 3  layout-content  What we set out to do (2–3 bullets)
Slide 4  layout-title    "The Problem"
Slide 5  layout-content  What wasn't working — lead with data
Slide 6  layout-two-col  Options considered vs option chosen
Slide 7  layout-title    "What We Built"
Slide 8  layout-content  Key features / approach (max 4 bullets)
Slide 9  layout-blank    Screenshot, diagram, or demo image
Slide 10 layout-content  Results — numbers first, then context
Slide 11 layout-content  Next steps — owner + deadline per item
Slide 12 layout-quote    One sentence summary / key takeaway
```

## Product Introduction

Structure: Hook → Pain → Solution → Proof → CTA

```
Slide 1  layout-cover    Product name + one-line value prop
Slide 2  layout-quote    A real user pain point, verbatim
Slide 3  layout-content  Why existing solutions fall short
Slide 4  layout-title    "Introducing [Product]"
Slide 5  layout-blank    Product screenshot or hero image
Slide 6  layout-two-col  Feature A vs Feature B (or Before/After)
Slide 7  layout-content  3 key benefits — outcome-focused, not feature-focused
Slide 8  layout-content  Social proof: numbers, logos, quotes
Slide 9  layout-cover    CTA — one clear action, large text
```

## Technical Talk / Engineering Share

Structure: Problem → Approach → Implementation → Learnings

```
Slide 1  layout-cover    Talk title + speaker name
Slide 2  layout-content  Agenda — 3–4 topics, no more
Slide 3  layout-title    "The Problem"
Slide 4  layout-content  Constraints and requirements
Slide 5  layout-code     Current state (bad code / architecture)
Slide 6  layout-title    "Our Approach"
Slide 7  layout-two-col  Options considered — tradeoffs
Slide 8  layout-code     Key implementation detail
Slide 9  layout-code     Another code example if needed
Slide 10 layout-content  Results — performance numbers, before/after
Slide 11 layout-content  What we'd do differently
Slide 12 layout-content  Key takeaways (3 bullets max)
Slide 13 layout-cover    Questions? + your contact / repo link
```

## Learning / Tutorial

Structure: Why → What → How → Practice → Summary

```
Slide 1  layout-cover    Topic + audience level (Beginner / Intermediate)
Slide 2  layout-content  What you'll learn today (3 outcomes)
Slide 3  layout-title    "Why This Matters"
Slide 4  layout-content  Real-world use cases
Slide 5  layout-title    "Core Concept"
Slide 6  layout-content  Definition + analogy
Slide 7  layout-code     Simplest possible example
Slide 8  layout-code     Slightly more realistic example
Slide 9  layout-two-col  Common patterns vs anti-patterns
Slide 10 layout-content  Gotchas and edge cases
Slide 11 layout-content  Summary — 3 things to remember
Slide 12 layout-cover    Resources + next steps
```

## Investor / Pitch Deck

Structure: Hook → Problem → Market → Solution → Traction → Ask

```
Slide 1  layout-cover    Company name + one-line pitch
Slide 2  layout-quote    The problem — make it visceral
Slide 3  layout-content  Market size (TAM / SAM / SOM)
Slide 4  layout-title    "Our Solution"
Slide 5  layout-blank    Product demo / screenshot
Slide 6  layout-two-col  Why us vs alternatives
Slide 7  layout-content  Traction — GMV, users, growth rate
Slide 8  layout-content  Business model — how you make money
Slide 9  layout-content  Team — names, relevant background only
Slide 10 layout-content  The ask — amount, use of funds
Slide 11 layout-cover    Thank you + contact
```

## Tips for all scenarios

- **Put the number big.** If a slide has one metric, make it 80–96px.
- **Every section needs a `layout-title` divider.** It gives the audience a moment to reset.
- **Don't end on "Q&A".** Close with your key message, then take questions.
- **Rehearse by reading only the slide titles.** If the story makes sense from titles alone, your structure is solid.
